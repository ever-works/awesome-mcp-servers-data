# GetAccept MCP Server

**Category:** Business & Commerce MCP Servers  
**Brand:** GetAccept  
**Slug:** `getaccept-mcp-server`

## Overview
GetAccept MCP Server is an integration that exposes GetAccept’s digital sales room and e-signature platform to AI agents and workflows. It allows applications to interact with GetAccept for managing documents, e-signatures, and sales engagement data via a standard MCP-compatible server hosted by Pipedream.

- **MCP server base URL:** `https://mcp.pipedream.net/v2`

## Features
- **MCP-compatible server**
  - Provides a static MCP server URL that can be used from any compatible client.
  - Centralized endpoint (`https://mcp.pipedream.net/v2`) for interacting with the GetAccept integration.

- **GetAccept digital sales room integration**
  - Connects AI agents and workflows to GetAccept’s digital sales room platform.
  - Intended for managing the lifecycle from sales opportunity to signed deal.

- **Document & e-signature workflows**
  - Designed to support management of documents and e-signature processes through MCP-enabled clients.
  - Enables automation around sending, tracking, and finalizing digital agreements (as supported by GetAccept’s platform).

- **Sales engagement data access**
  - Allows workflows and agents to interact with sales engagement data stored in GetAccept (e.g., engagement around proposals and contracts), subject to the underlying platform’s capabilities.

- **Client-agnostic integration**
  - The same MCP server URL can be used across different chat or agent clients.
  - Authentication is handled when adding the server to each application.

- **Pipedream-hosted**
  - Powered by Pipedream Connect infrastructure.
  - Central configuration and onboarding via Pipedream’s configuration pages.

## Setup
- Use the static MCP server URL: `https://mcp.pipedream.net/v2`.
- Add the server in your MCP-compatible chat or agent client.
- Authenticate within the client when prompted (details depend on the specific client and are handled during setup).
