# Venly MCP Server

MCP Server for Venly’s NFT API, enabling Web3 and NFT operations like minting and wallet management through the MCP ecosystem.

- **Website / Source**: https://mcp.pipedream.com/app/venly
- **Category**: Blockchain & Crypto MCP Servers
- **Brand**: Venly
- **MCP Server URL**: `https://mcp.pipedream.net/v2`

## Overview
The Venly MCP Server exposes Venly’s NFT API into the Model Context Protocol (MCP) ecosystem, allowing applications and AI agents to perform Web3 and NFT-related actions such as minting and wallet management via a unified MCP-compatible endpoint.

## Features
- **Unified MCP Endpoint**  
  - Single static MCP server URL: `https://mcp.pipedream.net/v2`  
  - Works with any compatible MCP client / chat application.

- **Venly NFT API Integration**  
  - Access to Venly’s NFT-focused Web3 API through MCP.  
  - Designed to support NFT-related operations (e.g., minting, wallet interactions) via Venly’s infrastructure.

- **Web3 & NFT Operations**  
  - Supports Web3 workflows for NFTs, including typical actions like minting and wallet management (as described in the item metadata).  
  - Suitable for onboarding Web2-style applications and users into Web3 by abstracting underlying blockchain complexity.

- **Wallet Management**  
  - Interface to Venly wallet capabilities through the MCP server (such as creating or managing wallets, per the Venly NFT API focus).

- **MCP Ecosystem Compatibility**  
  - Can be added to MCP-compatible chat clients / applications.  
  - Authentication is handled when adding the server to the application, not by changing the URL.

- **SaaS-Based Web3 Access**  
  - Built on Venly’s Web3 SaaS stack, emphasizing secure and easy-to-use software products for Web3 onboarding.

## Integration & Usage
- **Server URL**: Add `https://mcp.pipedream.net/v2` as an MCP server in your client or application.  
- **Authentication**: Performed during server setup in the client (details depend on the specific MCP client).  
- **Configuration Docs**: A general Configuration page is referenced (`/configuration`) for learning how to add MCP servers to supported chat clients.
