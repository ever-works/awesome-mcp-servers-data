# LogSnag MCP Server

**Category:** Monitoring  
**Tags:** event-tracking, notifications

An MCP server for LogSnag that enables simple event tracking and notifications within MCP-compatible workflows and tools.

---

## Overview
LogSnag MCP Server integrates LogSnag with MCP-based clients and tools, allowing you to send event data and trigger notifications through a standardized MCP server endpoint.

**MCP Server URL:**
```text
https://mcp.pipedream.net/v2
```
This is a static URL used by all supported MCP clients. Authentication is performed when you add/configure the server in your application.

---

## Features
- **Event tracking integration** with LogSnag for MCP workflows and tools.
- **Notifications support** for events tracked via LogSnag.
- **Standard MCP server endpoint** (`https://mcp.pipedream.net/v2`) usable across multiple MCP-compatible clients.
- **Client-agnostic setup**: same server URL works for every client; you authenticate per application.
- **Configuration guidance** available via a dedicated configuration page (for supported chat clients and tools).

---

## Setup
1. Use the static MCP server URL:
   ```text
   https://mcp.pipedream.net/v2
   ```
2. Add the server to your MCP-compatible app or chat client.
3. Authenticate within your application as prompted.
4. Optionally, consult the configuration documentation for client-specific steps.
