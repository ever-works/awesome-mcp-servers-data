# Onbee.app MCP Server

## Overview
The Onbee.app MCP Server connects Onbee’s HR-focused capabilities—such as onboarding, offboarding, digital employee cards, and HRIS data—into MCP-compatible workflows via Pipedream.

- **Category:** Business & Commerce – MCP Servers
- **Use Cases:** HR process automation, employee lifecycle management, integration of HRIS data into chat or MCP-based tools
- **MCP Server URL:** `https://mcp.pipedream.net/v2` (static URL used for all clients; authentication handled when adding the server to your application)

## Features
- **Onboarding & Offboarding Integration**
  - Integrate Onbee.app onboarding and offboarding workflows into MCP-compatible applications.
  - Use MCP tools to trigger or manage HR lifecycle events.

- **Digital Employee Cards**
  - Access or manage digital employee card data from Onbee.app within MCP workflows.

- **HRIS Data Connectivity**
  - Connect to Onbee.app as an HRIS-related data source for use in automation and chat-based workflows.

- **Static MCP Endpoint**
  - Single, static MCP server URL (`https://mcp.pipedream.net/v2`) for all clients.
  - Authentication occurs when you add the server to your chosen application or chat client.

- **Client-Agnostic Integration**
  - Can be added to various MCP-compatible chat clients and applications.
  - Configuration guidance is provided per client type via the Pipedream interface.

- **Configuration via Pipedream**
  - Connect your Onbee.app account and select a client to begin.
  - Central configuration and management of the MCP server within Pipedream.

## Setup
1. Connect your Onbee.app account in Pipedream.
2. Copy the MCP server URL: `https://mcp.pipedream.net/v2`.
3. Add the server to your MCP-compatible app or chat client.
4. Authenticate when prompted in your application.

For more detailed client-specific setup, refer to the Pipedream configuration documentation.
