# Kenjo MCP Server

HR software for deskless teams exposed via the Model Context Protocol (MCP) for automated HR workflows.

## Overview

The Kenjo MCP Server connects Kenjo’s HR capabilities to MCP-compatible clients, enabling automated and assisted HR workflows directly from chat or other MCP-aware applications.

- **Type:** MCP server / integration
- **Protocol:** Model Context Protocol (MCP)
- **Provider:** Kenjo (via Pipedream Connect)
- **Use case focus:** HR and workforce management for deskless teams

## MCP Server URL

Use this static URL with any compatible MCP client:

```text
https://mcp.pipedream.net/v2
```

Authentication is performed when you add the server to your application.

## Features

- **MCP-compatible HR integration**
  - Exposes Kenjo HR software functions over the MCP protocol.
  - Designed for use with chat-based or other MCP-aware clients.

- **Static server endpoint**
  - Single URL (`https://mcp.pipedream.net/v2`) works for all supported MCP clients.
  - Centralized configuration point for connecting Kenjo via MCP.

- **Authentication at client setup**
  - Authentication handled when adding the server in your chosen MCP client (exact method depends on the client and Pipedream Connect configuration).

- **Pipedream Connect integration**
  - Server is hosted and powered by Pipedream Connect.
  - Can be managed alongside other Pipedream-powered MCP servers and integrations.

- **Support for deskless-team HR workflows**
  - Oriented toward HR and workforce management scenarios typical for deskless employees (e.g., shift-based workforces), via MCP-driven automations.
