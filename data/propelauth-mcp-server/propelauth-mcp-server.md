# PropelAuth MCP Server

Team-based authentication for B2B SaaS within MCP workflows.

## Overview
The PropelAuth MCP Server is an MCP-compatible endpoint, powered by Pipedream Connect, that integrates PropelAuth’s team-based authentication and authorization into applications using the Model Context Protocol (MCP). It is accessed via a single static URL that can be used across compatible MCP clients.

- **Category:** Security / Attestation MCP Servers  
- **Provider / Brand:** PropelAuth (hosted via Pipedream Connect)  
- **MCP Server URL:** `https://mcp.pipedream.net/v2`

## Features
- **Team-based authentication for B2B SaaS**  
  - Designed for business-to-business applications that need multi-user, team- or organization-based access.  
- **Authorization for MCP workflows**  
  - Integrates PropelAuth’s authorization model into workflows that rely on MCP, allowing secured access to tools and resources.  
- **Static MCP server URL**  
  - Uses a single, static endpoint (`https://mcp.pipedream.net/v2`) that works for all supported MCP clients.  
  - Authentication is performed when the server is added to the client / application.  
- **Client-agnostic setup**  
  - The same server URL can be configured in multiple chat or MCP-compatible clients.  
- **Configuration documentation**  
  - A dedicated configuration page (on Pipedream) describes how to add and configure the server for different clients.

## Integration & Usage
- Add the MCP server to your preferred MCP-compatible chat client using the static URL.  
- Complete authentication during setup in the client.  
- Refer to the external configuration guide for step-by-step client-specific instructions.
