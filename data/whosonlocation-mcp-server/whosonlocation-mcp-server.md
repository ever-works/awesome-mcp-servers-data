# WhosOnLocation MCP Server

An MCP server integration that connects WhosOnLocation to compatible chat or MCP-enabled clients, enabling programmatic control of workplace presence and safety workflows.

## Overview

The WhosOnLocation MCP Server (by Pipedream) exposes WhosOnLocation capabilities through a static MCP endpoint so you can manage workplace sign-in and related operations directly from supported applications.

- **Category:** Business & Commerce – MCP Servers
- **Use cases:** Workplace sign-in, visitor and contractor management, desk booking, wellness/safety workflows
- **MCP Server URL:** `https://mcp.pipedream.net/v2`

## Features

- **Programmatic workplace sign-in**  
  Track and manage onsite employees, visitors, and contractors via MCP-enabled tools.

- **Touchless sign-in flows**  
  Support for contactless sign-in processes to reduce manual check-in and improve safety.

- **Desk booking**  
  Integrate with WhosOnLocation desk booking so users can reserve desks via MCP-compatible apps.

- **Visitor and contractor management**  
  Access WhosOnLocation’s visitor and contractor presence data and workflows from your chat or MCP client.

- **Wellness checks**  
  Incorporate wellness / health check steps into sign-in and presence workflows.

- **Central MCP endpoint**  
  Uses a single static MCP server URL (`https://mcp.pipedream.net/v2`) for all clients; authentication happens when adding the server to your application.

- **Client-agnostic integration**  
  Designed to be added to multiple chat / MCP clients; configuration instructions are provided per client type from the Pipedream interface.

- **Configuration via Pipedream**  
  Connect a WhosOnLocation account through Pipedream, select the client, and configure tools/actions exposed by the MCP server.
