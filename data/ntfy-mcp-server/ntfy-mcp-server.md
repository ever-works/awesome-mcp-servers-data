## Overview

ntfy MCP Server exposes the ntfy push notification service through the Model Context Protocol (MCP). It allows MCP-enabled clients and workflows to send push notifications to mobile devices using simple HTTP methods (PUT/POST) via a static MCP server endpoint.

Source: <https://mcp.pipedream.com/app/ntfy>

## Features

- **MCP-based push notifications**
  - Integrates ntfyâ€™s push notification service with MCP-compatible clients.
  - Allows sending notifications to phones through MCP-operated workflows.

- **Simple HTTP interface**
  - Supports sending notifications using `PUT` and `POST` HTTP methods.
  - Designed for straightforward integration from automations, tools, or agents.

- **Static MCP server URL**
  - Uses a single, static MCP server endpoint: `https://mcp.pipedream.net/v2`.
  - Same URL works for all compatible clients.

- **Client-agnostic configuration**
  - Can be added to different MCP-capable chat clients or applications.
  - Authentication occurs when adding/configuring the server in the client.

- **Configuration resources**
  - Additional setup and usage details available via a dedicated configuration page (linked from the app page).
