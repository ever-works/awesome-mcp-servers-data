# OneSignal MCP Server (User Auth)

## Overview
The OneSignal (User Auth) MCP Server exposes OneSignal push messaging capabilities through the Model Context Protocol (MCP) with user authentication support. It is provided as a static MCP endpoint that can be added to compatible chat clients or MCP-enabled applications.

- **Category:** Messaging MCP Servers  
- **Brand:** OneSignal  
- **Use cases:** Push notifications, messaging, user engagement via MCP-compatible clients

## Connection Details
- **MCP Server URL:** `https://mcp.pipedream.net/v2`  
- This is a **static URL** that works for all clients; authentication is performed when adding/configuring the server in your application.

## Features
- **Model Context Protocol (MCP) access** to OneSignal push messaging capabilities.
- **User authentication support** for secure access to push messaging features.
- **Static MCP endpoint** (`https://mcp.pipedream.net/v2`) usable across multiple MCP clients.
- **Client-agnostic setup:** can be added to various MCP-compatible chat clients or applications.

## Setup
- Copy and use the static MCP server URL: `https://mcp.pipedream.net/v2`.
- Add the server to your MCP-compatible chat client or application.
- Complete user authentication during the server-add / configuration flow in your client.
