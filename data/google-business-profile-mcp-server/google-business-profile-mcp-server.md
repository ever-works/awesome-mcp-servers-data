# Google Business Profile MCP Server

## Overview
The Google Business Profile MCP Server is an MCP (Model Context Protocol) integration that connects Google Business Profile (formerly Google My Business) with MCP-enabled applications. It helps manage business visibility and customer engagement across Google Search and Maps, including support for multiple locations.

## Features
- **Google Business Profile integration**  
  - Connects MCP tools and chat clients to Google Business Profile.  
  - Works across Google Search and Google Maps surfaces.

- **Business visibility & engagement**  
  - Supports showing up when customers search for businesses on Google Search and Maps.  
  - Leverages the latest engagement features of Google Business Profile to stay connected with customers (e.g., profile updates, interactions; specific tools are loaded dynamically in the UI).

- **Multi-location management**  
  - Designed to manage multiple business locations from a single MCP server integration.

- **Account configuration**  
  - Connect a Google account to configure access to Google Business Profile.  
  - Select a specific client/profile after connection to begin managing it.

- **Static MCP endpoint**  
  - Uses a single static server URL (`https://mcp.pipedream.net/v2`) for all clients.  
  - Authentication occurs when adding the server to the MCP-compatible application, not by changing the URL per client.

- **Client-agnostic setup**  
  - Can be added to different chat clients or MCP-compatible applications.  
  - Documentation available via a centralized configuration page on Pipedream.

## Setup & Configuration
- Connect your Google account through the Pipedream interface.  
- Select the desired Google Business Profile client/account.  
- Copy the static MCP server URL: `https://mcp.pipedream.net/v2`.  
- Add the server URL to your MCP-compatible app or chat client and authenticate there.  
- Use the available tools (loaded dynamically) to manage locations and customer engagement.
