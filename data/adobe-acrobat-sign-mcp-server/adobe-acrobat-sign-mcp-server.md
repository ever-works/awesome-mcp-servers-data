# Adobe Acrobat Sign MCP Server

## Overview
The Adobe Acrobat Sign MCP Server is an MCP-compatible integration that exposes Adobe Acrobat Sign’s e‑signature and digital signing capabilities to AI tools and workflows via Pipedream. It lets you connect Adobe Acrobat Sign to MCP-enabled chat clients and applications using a static server URL.

- **Category:** Business & Commerce – MCP Servers  
- **Provider / Brand:** Adobe (via Pipedream)  
- **Primary Use Cases:** E‑signatures, digital signing, document workflows, contracts, surveys, and forms

## Features
- **MCP-compatible e‑signature server**  
  - Provides an MCP Server endpoint that can be added to MCP-capable chat clients and applications.  
  - Enables AI workflows to trigger and interact with Adobe Acrobat Sign capabilities.

- **Static server URL**  
  - Single, static MCP server URL for all clients: `https://mcp.pipedream.net/v2`.  
  - Simplifies configuration across multiple tools and environments.

- **Authentication at application level**  
  - Authentication occurs when adding the MCP server to your application, allowing secure connection of your Adobe Acrobat Sign account.

- **Adobe Acrobat Sign integration**  
  - Utilizes Adobe Acrobat Sign’s e‑signature and digital signing features.  
  - Suitable for workflows involving contracts, document approvals, surveys, and forms.

- **Multi‑client compatibility**  
  - Can be added to different MCP-enabled chat clients or applications.  
  - Configuration guidance available per client type (via the “Add the server to your app” flow and configuration page).

- **Configuration guidance**  
  - Central configuration page (linked from the app) to help set up the server and tools in various clients.
