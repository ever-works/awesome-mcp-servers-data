### Overview

**mcp.run** is a hosted registry and control plane for installing and running secure, portable MCP servers. It enables centralized management of MCP deployments, with options for both self-hosted and cloud-ready setups, giving teams strict control over where and how agent traffic flows.

### Features

- **Self-hosted & cloud-ready deployment**  
  - Can be deployed into your own environment.  
  - Supports keeping agent traffic behind your firewall.  
  - Provides full control over data flow to reduce risks of data exfiltration or exposure to third-party vulnerabilities.

- **Centralized MCP server registry**  
  - Maintain an internal, trusted registry of MCP servers.  
  - Organize and provide only the servers your team needs.

- **Role-based access control (RBAC)**  
  - Admins can approve and manage which MCP servers are available to which users or teams.  
  - Helps enforce organizational policies around tool and data access.

- **Audit logging & activity monitoring**  
  - Fully audited admin logs for tracking MCP-related activity.  
  - Visibility into how agents interact with MCP servers and connected resources.

- **Centralized access control / kill-switch**  
  - Ability to deactivate agent access to data and applications from a central control plane.  
  - Designed to restrict or revoke agent access without disrupting direct human access to work applications.

- **Security-focused design**  
  - Emphasis on secure, portable MCP servers.  
  - Architecture aimed at minimizing third-party dependency risk while keeping agents effective within controlled environments.
