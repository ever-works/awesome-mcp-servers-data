# San Francisco Open Data MCP Server

## Overview
The **San Francisco Open Data – DataSF MCP Server** provides access to open data from the City and County of San Francisco via the Model Context Protocol (MCP). It connects to the DataSF platform, enabling search and retrieval across hundreds of public datasets within compatible MCP-enabled applications (such as chat or AI clients).

- **Category:** Data Access & Integration MCP Servers  
- **Provider / Brand:** DataSF (via Pipedream)  
- **Use cases:** Open data exploration, analytics, research, and application integration with San Francisco city datasets.

## Features
- **MCP-compatible server endpoint**  
  - Static MCP server URL: `https://mcp.pipedream.net/v2`  
  - Single URL works across supported MCP clients.

- **Access to San Francisco open data**  
  - Interfaces with the DataSF platform.  
  - Search and access across hundreds of datasets from the City and County of San Francisco.  
  - Suitable for analytics and data exploration use cases.

- **Account-based configuration**  
  - Connect your account through Pipedream.  
  - Authenticate when adding the server to your MCP-compatible application.

- **Client-agnostic integration**  
  - Designed to be added to multiple chat or AI clients that support MCP.  
  - Central configuration instructions available via the platform’s configuration page.

- **Tool discovery within clients**  
  - Exposes “available tools” (actions) that can be loaded within the MCP client to interact with DataSF datasets (e.g., searching, querying).  
  - Tools are dynamically loaded by the client after the server is connected.

## Setup
- Copy and use the MCP server URL: `https://mcp.pipedream.net/v2`.  
- Add the server to a supported MCP client and authenticate when prompted.  
- Optionally refer to the platform’s full configuration page for client-specific setup instructions.
