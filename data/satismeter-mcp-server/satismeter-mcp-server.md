# SatisMeter MCP Server

Customer feedback MCP server integration for SatisMeter, exposing survey and NPS capabilities to MCP-enabled applications.

## Overview
The SatisMeter MCP Server integrates SatisMeter’s customer feedback and survey functionality (including Net Promoter Score measurement) into MCP-enabled apps via a static MCP server endpoint provided by Pipedream.

- **Category:** Business & Commerce – MCP Servers  
- **Vendor / Brand:** SatisMeter (via Pipedream Connect)  
- **Integration Type:** MCP Server (Model Context Protocol)  
- **MCP Server URL:** `https://mcp.pipedream.net/v2`

## Features
- **In-app surveys**  
  - Trigger and manage surveys directly inside your application’s interface (via MCP integration with SatisMeter).

- **Email surveys**  
  - Initiate and manage email-based surveys to collect customer feedback.

- **NPS (Net Promoter Score) measurement**  
  - Run NPS surveys to measure customer loyalty and satisfaction.  
  - Use MCP-enabled tools to surface and act on NPS data provided by SatisMeter.

- **Customer feedback collection**  
  - Collect structured feedback from users through in-app and email channels.  
  - Centralize survey-driven insights within MCP-compatible clients.

- **Product and customer insights**  
  - Use survey responses to understand how customers interact with your product and business.  
  - Enable downstream analysis and workflows through MCP-enabled applications.

- **Static MCP server endpoint**  
  - Single static URL (`https://mcp.pipedream.net/v2`) used for all clients.  
  - Authentication is handled when adding the server to each client / application.

- **Multi-client compatibility**  
  - Designed to be added to various MCP-compatible chat or agent clients.  
  - Configuration guidance available via a generic configuration page (not client-specific in this summary).

## Setup
- Use the static MCP server URL: `https://mcp.pipedream.net/v2` when configuring the SatisMeter MCP Server in your MCP-enabled application or chat client.
- Authenticate within your client when adding the server (authentication details are handled by the client’s configuration flow).
