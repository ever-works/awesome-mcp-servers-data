# Radar MCP Server

## Overview
Radar MCP Server is an MCP-compatible integration that connects the Radar geofencing platform to MCP clients, enabling applications to build and use location-based and geofencing functionality via Radar APIs. It is provided as a static MCP server endpoint that you connect to from compatible chat or agent clients.

- **Category:** Data Access & Integration – MCP Servers  
- **Provider / Brand:** Radar (via Pipedream Connect)  
- **MCP Server URL:** `https://mcp.pipedream.net/v2`

## Features
- **MCP-compatible server** – Exposes Radar functionality through the Model Context Protocol so MCP-capable clients can call Radar APIs.
- **Static server endpoint** – Uses a single static URL (`https://mcp.pipedream.net/v2`) for all supported clients.
- **Radar API integration** – Connects to Radar’s location and geofencing APIs to enable location-aware features in your applications.
- **Authentication at client setup** – You authenticate to Radar when adding the MCP server to your application/client (per-client auth, shared URL).
- **Works with multiple chat/agent clients** – Designed to be added to various MCP-enabled chat clients; setup instructions are client-specific.
- **Configuration documentation** – Additional configuration details available via the referenced Configuration page on the hosting site.

## Use Cases
- Building location-based product features that rely on Radar APIs.
- Integrating geofencing behavior into MCP-aware chatbots or agent applications.
- Enabling location intelligence workflows within tools that support MCP servers.

## Technical Details
- **Protocol:** Model Context Protocol (MCP)
- **Server type:** Hosted, shared static endpoint
- **URL:** `https://mcp.pipedream.net/v2`
- **Host platform:** Pipedream Connect
