# GeoDB Cities MCP Server

## Overview
GeoDB Cities MCP Server is an MCP-compatible integration for the GeoDB Cities API. It allows MCP-enabled applications and chat clients to search for and retrieve structured data about cities and towns around the world via a standardized MCP server endpoint.

- **Category:** Data Access & Integration – MCP Servers
- **Provider / Brand:** GeoDB (via Pipedream)
- **Slug:** `geodb-cities-mcp-server`
- **Primary Endpoint (MCP server URL):** `https://mcp.pipedream.net/v2`

## Features
- **Global city and town search**
  - Access the GeoDB Cities API through an MCP server interface.
  - Search for cities and towns worldwide.

- **MCP-compatible server**
  - Exposes GeoDB Cities functionality as an MCP server usable by MCP-compatible tools and chat clients.
  - Uses a static MCP server URL (`https://mcp.pipedream.net/v2`) for all supported clients.

- **Simple configuration**
  - Connect your account through Pipedream’s configuration flow.
  - Select your client and follow per-client setup instructions.
  - Authentication occurs when adding the server to your application, not when copying the URL.

- **Client-agnostic setup**
  - Same MCP server URL works across multiple chat clients.
  - Documentation available via a central configuration page on Pipedream.

## Integration & Usage
- **How it’s used**
  - Add `https://mcp.pipedream.net/v2` as an MCP server in your MCP-capable app or chat client.
  - Authenticate when prompted by your client / Pipedream.
  - Use the exposed MCP tools to look up and retrieve city and town data from GeoDB Cities.
