# Nutrient Document Web Services API MCP Server

## Overview
The **Nutrient Document Web Services API MCP Server** is an MCP (Model Context Protocol) server that exposes Nutrient’s PDF and document processing APIs through a single endpoint, enabling AI-driven document workflows. It provides access to over 25 PDF tools via MCP-compatible applications.

- **Category:** Document Management MCP Servers  
- **Provider / Brand:** Nutrient  
- **Integration host:** Pipedream  
- **Primary use case:** AI and application workflows that need PDF and document processing via MCP

## Features
- **MCP-compliant server**
  - Exposes Nutrient Document Web Services APIs through the Model Context Protocol.
  - Usable from MCP-compatible chat and AI clients.

- **Centralized API endpoint**
  - Static MCP server URL: `https://mcp.pipedream.net/v2`.
  - Same URL for all clients; authentication occurs when adding the server to an application.

- **PDF and document processing tools**
  - Access to **25+ PDF tools** (exact tools not listed in the source content, but collectively described as PDF and document processing APIs).
  - Designed for workflows such as PDF handling and document transformation through AI agents.

- **Fast setup & configuration**
  - “Fast setup” for connecting Nutrient Document Web Services API to MCP-based clients.
  - Configuration flow to connect an account and select a client.
  - Guided instructions per chat client (e.g., “Select your chat client to learn how to get started”).
  - Link to a full configuration page for more detailed setup steps.

- **Client-agnostic usage**
  - Single MCP server URL works across multiple chat / AI clients.
  - Designed to be added as a server within compatible applications for AI-driven document workflows.
