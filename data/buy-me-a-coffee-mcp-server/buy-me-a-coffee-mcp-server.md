# Buy Me a Coffee MCP Server

## Overview
The Buy Me a Coffee MCP Server lets MCP-compatible agents interact with Buy Me a Coffee creator-support capabilities—such as donations, memberships, and digital product sales—through a standardized MCP endpoint.

- **Type:** MCP Server (integration endpoint)
- **Category:** Business & Commerce – MCP Servers
- **Brand:** Buy Me a Coffee
- **MCP Endpoint URL:** `https://mcp.pipedream.net/v2`

## Features
- **Unified MCP Endpoint**  
  - Uses a static MCP server URL (`https://mcp.pipedream.net/v2`) that works across clients.  
  - Authentication is performed when you add the server to your MCP-compatible application or chat client.

- **Creator Support Operations (via MCP)**  
  - Integrates with Buy Me a Coffee concepts such as:  
    - Donations / one-time support.  
    - Memberships / recurring support.  
    - Selling digital items or other offerings.  
  - Exposes these capabilities to MCP agents so they can work with creator-support workflows programmatically.

- **Client-Agnostic Integration**  
  - Designed to connect with multiple MCP-enabled chat clients or agent frameworks.  
  - Uses the same base URL for all supported clients.

- **Configuration Guidance**  
  - Documentation available through a dedicated configuration page (linked as “Configuration page”) explaining how to add the server to different chat clients.

## Technical Details
- **Protocol:** Model Context Protocol (MCP) server.  
- **Host Platform:** Implemented and hosted via Pipedream Connect.  
- **Access Method:** Add as an MCP server in your client using the static URL and perform authentication within that client.
