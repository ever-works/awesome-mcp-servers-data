# Voiceflow MCP Server

An MCP (Model Context Protocol) server that lets you design and run chat or voice assistants using Voiceflow’s collaborative platform inside any MCP-compatible client.

- **Name:** Voiceflow MCP Server  
- **Category:** AI Integration – MCP Servers  
- **Brand:** Voiceflow  
- **Source URL:** https://mcp.pipedream.com/app/voice  
- **MCP Server URL:** `https://mcp.pipedream.net/v2`

## Overview
The Voiceflow MCP Server exposes Voiceflow’s assistant design and runtime capabilities through a unified MCP endpoint, allowing MCP-based agents and tools to integrate with assistants created in Voiceflow. The same static MCP URL is used across clients, with authentication handled when you add the server to your application.

## Features
- **Static MCP endpoint**  
  - Single, static URL (`https://mcp.pipedream.net/v2`) used across all supported MCP clients.
- **Voiceflow assistant integration**  
  - Connects MCP-based agents to assistants designed and managed in Voiceflow’s collaborative platform.  
  - Supports both chat and voice assistant use cases (based on Voiceflow’s core capabilities).
- **Client-agnostic configuration**  
  - Designed to work with any MCP-compatible chat client or application.  
  - Authentication is performed when adding the server into the chosen client.
- **Configuration documentation**  
  - Dedicated configuration page (linked as “Configuration page”) with setup details per client (selection of chat client and setup steps).
- **Hosted by Pipedream Connect**  
  - MCP server infrastructure provided and maintained by Pipedream Connect.

## Integration & Usage
- Add the MCP server to your MCP client using the static URL:  
  `https://mcp.pipedream.net/v2`
- Authenticate during the add / configuration step within your client.  
- Refer to the linked “Configuration page” for client-specific instructions.
