# Vitally MCP Server

Modern customer success data and automation for B2B SaaS, exposed via an MCP server for use by MCP-compatible agents and chat clients.

## Overview
- **Type:** MCP server (Model Context Protocol)
- **Purpose:** Expose Vitally B2B SaaS customer success data, analytics, and automation capabilities to MCP agents
- **Static MCP Server URL:** `https://mcp.pipedream.net/v2`
- **Provider / Host:** Pipedream Connect
- **Category:** Business & Commerce MCP servers
- **Tags:** customer-success, analytics, automation

## Features
- **Customer Success Data Access**  
  - Exposes Vitally customer success data to MCP agents  
  - Designed for B2B SaaS customer accounts and lifecycle data

- **Analytics Integration**  
  - Provides access to Vitally analytics within MCP-enabled tools  
  - Supports deeper understanding of unified customer data (e.g., health, usage, engagement) as exposed by Vitally

- **Automation Capabilities**  
  - Surfaces Vitallyâ€™s automation features to MCP agents  
  - Can be used to build workflows and automated actions around customer success operations

- **Unified Data Access**  
  - Intended to unify disparate customer success data sources managed in Vitally  
  - Makes this unified view accessible programmatically through MCP

- **Static Server Endpoint**  
  - Single static MCP server URL (`https://mcp.pipedream.net/v2`) for all clients  
  - Authentication performed when adding the server to each application

- **Client-Agnostic Setup**  
  - Works with multiple chat clients that support MCP  
  - Configuration guided via client-specific instructions and a general configuration page

## Integration & Configuration
- **Add to Chat Clients:**  
  - Select a compatible chat client and follow its specific instructions to add the Vitally MCP server
- **General Configuration Reference:**  
  - A full configuration page is available to help set up the MCP server (authentication, client settings, etc.)
